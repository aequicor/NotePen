# Corner Case Register — Main Screen Redesign

> **Сгенерировано на фазе бизнес-требований.**
> Critical и High — обязательные входные данные для спецификации и плана реализации.
> Spec ДОЛЖНА ссылаться на каждый Critical и High пункт.
> План ДОЛЖЕН включать тестовую задачу для каждого Critical пункта.

---

### Critical (потеря данных, безопасность, неработающий бизнес-процесс)

| # | Категория | Условие | Ожидаемое бизнес-поведение | Spec Addressed |
|---|-----------|---------|---------------------------|----------------|
| CC-1 | Input Integrity | SAF URI стал невалидным после реавторизации SAF; fuzzy-match предлагает объединить; пользователь ошибочно отклоняет объединение → создаётся дублирующая запись, оригинальная теряет актуальный URI | При отклонении объединения создаётся новая запись с новым URI; старая запись с устаревшим URI помечается «ошибка файла» и остаётся в списке (retain-with-retry). Дублей не возникает — обе записи сосуществуют как отдельные. | ❌ |
| CC-2 | Input Integrity | `displayName` и размер совпадают у двух разных файлов → fuzzy-match ложно предлагает объединить разные документы | При ложном срабатывании fuzzy-match пользователь должен иметь возможность видеть полные пути/URI обоих файлов перед принятием решения об объединении. Если пользователь отклоняет — обе записи остаются отдельно. | ❌ |
| CC-5 | Business Process Integrity | Flush истории успешен (AC-18), но приложение крашнулось между flush и показом редактора → при следующем запуске файл есть в истории, но незавершённый сеанс редактирования отсутствует | При следующем запуске файл корректно присутствует в истории с актуальным статусом доступности. Незавершённый редакторный сеанс не восстанавливается (вне скоупа). | ❌ |
| CC-8 | Domain Invariants | AC-9e требует вытеснить самую старую **недоступную** запись; но асинхронная проверка доступности (AC-9a) ещё не завершилась — статус записи «неизвестен». Алгоритм вытеснения видит все 20 записей как «ещё не проверены» | При вытеснении записи с неизвестным статусом она трактуется как недоступная (pessimistic). Если после завершения проверки выясняется, что вытесненная запись была доступна — восстановление не требуется (eventual consistency по AC-9a). | ❌ |

### High (заметная некорректность, сломанный пользовательский сценарий)

| # | Категория | Условие | Ожидаемое бизнес-поведение | Spec Addressed |
|---|-----------|---------|---------------------------|----------------|
| CC-3 | Input Integrity | Путь к файлу содержит символы Unicode (кириллица, иероглифы, emoji); canonical path нормализация может вести себя по-разному на разных ОС и FS | Нормализация пути применяет Unicode NFC перед сравнением на всех платформах. Если нормализация невозможна — используется строгое побайтовое сравнение без нормализации (безопаснее ложного дублирования, чем ложного объединения). | ❌ |
| CC-6 | Business Process Integrity | Пользователь открывает файл из диалога; до завершения flush истории (AC-18) нажимает системную кнопку «Назад» — навигация в редактор отменяется | Flush истории отменяется вместе с отменой навигации. Если flush уже выполнен — запись удаляется из истории (откат). | ❌ |
| CC-7 | Business Process Integrity | Двойное нажатие (double-tap) на элемент списка → возможен запуск двух экземпляров редактора или двойная запись в историю | Повторный тап игнорируется пока навигация к редактору уже инициирована (debounce/single-launch protection). История пишется ровно один раз. | ❌ |
| CC-9 | Domain Invariants | История полная (20 доступных записей); пользователь открывает новый файл → AC-5 вытесняет самую старую запись; race condition: «самая старая» может быть файлом, только что добавленным | Вытеснение определяется по временной метке AC-23 (момент инициирования). Только что добавленный файл имеет самую свежую метку — не вытесняется. Race condition невозможен при корректной сортировке по timestamp. | ❌ |
| CC-10 | Domain Invariants | Desktop: два окна одновременно достигают лимита 20 и оба пытаются вытеснить самую старую запись | Операция вытеснения является атомарной для общей истории (AC-19); второй запрос видит уже обновлённое состояние. Итого: вытесняется ровно одна запись, добавляются обе новые — если лимит позволяет. | ❌ |
| CC-11 | External Dependency | PDF-движок выбрасывает OOM при генерации миниатюры для очень большого файла (500+ страниц, высокий DPI) | Генерация миниатюры выполняется в изолированном контексте; OOM перехватывается, элемент получает плейсхолдер ошибки (AC-9d). Приложение продолжает работать. Исключение логируется. | ❌ |
| CC-12 | External Dependency | SAF возвращает URI, но при открытии потока бросает `SecurityException` (разрешение истекло) | Запись переводится в состояние «ошибка файла» (AC-9c), не «файл не найден». Пользователь видит соответствующее сообщение. | ❌ |
| CC-15 | Temporal & Concurrency | Desktop: окно редактора закрыто (триггер AC-17) → главный экран обновляет список → но flush истории ещё не завершился (AC-18 non-blocking) → список обновился со старым статусом записи | Обновление списка на Desktop инициируется после завершения flush (или с небольшой задержкой/повторной проверкой). Если flush завершился с ошибкой — список показывает последнее известное состояние. | ❌ |
| CC-17 | User Journey | Новый пользователь видит пустое состояние (AC-1); кнопка открытия файла не очевидна в интерфейсе (OQ-6 открыт); пользователь не понимает, что делать | Пустое состояние должно явно указывать на действие: текстовая подсказка содержит описание кнопки открытия файла или указывает на её местоположение. @Designer учитывает этот сценарий при проектировании первого запуска. | ❌ |
| CC-18 | User Journey | Android: пользователь уходит из приложения (home) во время генерации миниатюр; ОС ограничивает CPU; при возврате список показывает плейсхолдеры бесконечно | При возврате приложения на передний план генерация миниатюр для видимых элементов возобновляется или перезапускается. Плейсхолдер не застревает. | ❌ |
| CC-19 | User Journey | Пользователь удалил файл внешним менеджером **пока главный экран открыт**; фоновая проверка не ведётся (AC-9a); запись выглядит доступной → пользователь нажимает → неожиданная ошибка | При попытке открыть файл из списка выполняется синхронная проверка доступности непосредственно перед открытием. Если файл недоступен — запись немедленно обновляется до нужного состояния, пользователь получает понятное сообщение. | ❌ |

### Medium (ухудшенный опыт, обходной путь существует)

| # | Категория | Условие | Ожидаемое бизнес-поведение | Spec Addressed |
|---|-----------|---------|---------------------------|----------------|
| CC-4 | Input Integrity | PDF-файл с размером 0 байт добавлен в историю; миниатюра не может быть сгенерирована | Плейсхолдер ошибки рендеринга (AC-9d). Файл можно попробовать открыть — получит «ошибка файла» (AC-9c). | ❌ |
| CC-13 | Scale | 20 файлов в истории; пользователь быстро прокручивает список → все 20 миниатюр генерируются параллельно → перегрузка thread pool | Генерация миниатюр ограничена фиксированным числом параллельных задач (concurrency limit), определяемым в спецификации. Видимые элементы имеют приоритет над скрытыми. | ❌ |
| CC-16 | Temporal | Файл изменён внешним редактором; FS не обновила mtime (FAT32, некоторые сетевые FS) → кеш не инвалидирован → показывается устаревшая миниатюра | Это known limitation; документируется в спецификации. Дополнительная проверка по размеру файла как fallback к mtime рассматривается как улучшение в будущем. | ❌ |
| CC-20 | User Journey | Android: смена ориентации (portrait ↔ landscape) пересекает breakpoint 600 dp → сетка меняется; позиция прокрутки и состояние генерации миниатюр должны сохраниться | При смене ориентации список сохраняет позицию прокрутки (первый видимый элемент). Уже сгенерированные миниатюры не перегенерируются. | ❌ |
| CC-21 | User Journey | Android: переустановка приложения инвалидирует все SAF content:// URI в истории → вся история помечается «ошибка файла» | Ожидаемое поведение; OQ-4 (резервное копирование) всё ещё открыт. При наличии backup-механизма URI могут быть восстановлены. Без backup — пользователь видит список с ошибками; может открыть файлы заново через диалог. | ❌ |

### Low (косметика, экстремальный edge)

| # | Категория | Условие | Ожидаемое бизнес-поведение | Spec Addressed |
|---|-----------|---------|---------------------------|----------------|
| CC-14 | Scale | Кеш LRU вытеснил миниатюры; при прокрутке те же 20 миниатюр перегенерируются повторно → постоянный кеш-miss при малом лимите | Лимит 50 МБ достаточен для типичного использования (20 записей × ~2.5 МБ на миниатюру). Снижение лимита до < 10 МБ не рекомендуется. | ❌ |
| CC-22 | User Journey | Пользователь с RTL-локалью (арабский, иврит); порядок элементов и кнопки могут некорректно зеркалироваться | Compose Multiplatform поддерживает LayoutDirection; @Designer учитывает RTL при разработке компонентов. | ❌ |

| CC-23 | Domain Invariants | Файл одновременно архивный (вытеснен из истории) И недоступен на диске | «Ошибка файла» доминирует в UI (AC-53); архивный статус сохраняется в модели. Попытка открыть → сообщение о недоступности. | ❌ |
